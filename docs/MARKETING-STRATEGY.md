### 1-1. Acquisition (획득) — 진단: D (채널 없음, 즉시 개선 필요)

**현황**

- 공식 런칭 마케팅 없이 제품만 배포된 상태
- GitHub Public 레포 오픈 (https://github.com/namseokyoo/pulse)
- SEO 기초 설정: layout.tsx에 OG, Twitter Card, robots 설정 완료
- 키워드: 커뮤니티, 익명 게시판, 실시간, 생명력, PulseUp, 펄스업

**강점**

- 독특한 컨셉(생명력 시스템)은 구전(입소문) 마케팅에 유리
- GitHub Public 레포 → 개발자 커뮤니티 유입 가능성
- 봇 시스템으로 신규 방문자 첫 진입 시 콘텐츠 공백 없음

**약점**

- 브랜드 SNS 계정 부재 (Twitter/X, Instagram 등)
- Product Hunt 등록 없음
- 국내 커뮤니티(네이버, DC인사이드, 에펨코리아 등) 마케팅 없음
- 타겟(20-35세) 도달 채널 전무

### 1-2. Activation (활성화) — 진단: B (기반 구축, 체감 레이어 개선 중)

**현황**

- 비로그인 사용자: 피드 읽기 가능 (공개 읽기 전용 피드 구현)
- 로그인 유도: "투표하고 글의 생명을 결정하세요" 배너
- 온보딩: Google OAuth 1단계 → 랜덤 닉네임 부여
- Phase 1.2 (UX 체감 강화) 완료 여부: PRD 기준 DoD 체크 미완료 항목 존재

**강점**

- 진입 장벽 낮음: Google OAuth 원클릭 가입
- 무료 투표권 10개/일 → 즉시 가치 경험 가능
- 비로그인도 피드 탐색 가능 → 컨셉 이해 후 가입 유도

**약점**

- "글에 생명력을 부여하다" 태그라인은 개념적 — 처음 보는 사용자가 3초 내 이해 어려움
- 정렬 탭: "최신순 / 생명력 적은순 / 생명력 많은순" → 세계관 문구 미반영 (PRD 1.2 예정)
- 온보딩 후 첫 행동 유도 부재 (글 쓰기? 투표하기? 안내 없음)
- 생명력 상태 배지 미구현 시 게임성 체감 어려움

**Aha Moment 정의**:
> 사용자가 처음 글에 투표했을 때 타이머가 실시간으로 변하는 것을 목격하는 순간

### 1-3. Retention (리텐션) — 진단: C (의존 구조 취약)

**현황**

- 무료 투표권 매일 정오 10개 충전 → 재방문 트리거 존재
- 투표권 리셋 카운트다운 UI 구현
- 알림 시스템 없음 (Phase 3 예정)
- 이메일 뉴스레터 없음

**강점**

- 투표권 매일 충전 → Daily Hook 메커니즘 내장
- 생존하는 글 보기 욕구 → 자연적 재방문 유도
- 봇 활동 → 매일 새로운 콘텐츠 공급 보장

**약점**

- 알림 없음 → "내 글이 위험해요" 같은 긴급 리텐션 불가
- 팔로우/친구 기능 없음 → 사회적 리텐션 없음
- DAU/MAU 측정 인프라 불명 (GA4 연동 여부 미확인)

**Hook Model 분석**:
- External Trigger: 없음 (알림, SNS 없음)
- Internal Trigger: "내 글이 살아있을까?" 호기심
- Variable Reward: 글의 생존/소멸 결과
- Investment: 글 작성, 투표권 소비

### 1-4. Revenue (수익) — 진단: B (구조 완성, 유입 부재)

**현황**

- Lemon Squeezy 결제 연동 완료 (Phase 2)
- 유료 투표권 3개 패키지 (2,000원 / 9,000원 / 16,000원)
- 마진율 우수: 58% ~ 90%

**강점**

- 결제 인프라 완성 → 즉시 수익 창출 준비 완료
- 무료→유료 전환 설계 명확 (무료 10개/일 → 부족하면 구매)
- 영구 보유 유료 투표권 → 구매 동기 명확

**약점**

- 현재 유입 사용자 수 불명 → MRR 추정 불가
- 유료 구매 페이지로의 유도 UI 강도 미확인
- 구매 완료 후 재구매 유도 메커니즘 불명

## 2. 봇 시스템의 마케팅적 의미 분석

### 봇 시스템 역할

10개 페르소나 봇이 30분/1시간/1.5시간 주기로 자동 콘텐츠 생성 중. 이는 순수 마케팅 관점에서 다음 의미를 갖는다.

| 기능 | 마케팅 효과 |
|------|------------|
| 글 작성 (30분) | 신규 방문자가 빈 피드를 보지 않음 → 첫인상 최적화 |
| 투표 (1시간) | 기존 글의 생명력 연장 → 다양성 있는 피드 유지 |
| 댓글 (1.5시간) | 참여감 형성 → "활발한 커뮤니티처럼" 보임 |

### 봇 시스템의 한계

- 봇 활동은 실제 사용자 Engagement 수치를 오염시킴 → 지표 해석 시 봇 트래픽 분리 필요
- 봇 페르소나 10개가 다양한 연령/직업 반영 → 콘텐츠 다양성은 우수
- 실제 사용자 리텐션 데이터를 오도할 수 있으므로, 봇 계정은 별도 필터링 후 분석해야 함

### 권장 지표 분리

> 봇 계정 이메일(@pulseup.internal 도메인)을 GA4 또는 Supabase 분석에서 제외하여 실제 사용자 지표만 추적할 것.

## 3. 마케팅 전략

### 3-1. 홈페이지 / 랜딩페이지 CRO 전략

#### 현재 홈페이지 상태 분석

현재 홈페이지(/)는 피드 자체가 랜딩페이지 역할을 겸하는 구조. 비로그인 사용자에게 작은 배너 1개로 온보딩 유도. 이는 전형적인 제품 우선형 설계로, 컨셉을 모르는 신규 방문자에게 불친절하다.

#### CRO 개선 제안 (우선순위순)

**P0: 3줄 핵심 룰 배너로 교체 (Phase 1.2 DoD 항목)**

현재 비로그인 사용자 배너:
- "투표하고 글의 생명을 결정하세요"
- "좋아요는 +10분, 싫어요는 -10분의 생명력을 변화시킵니다"

이를 3줄 핵심 룰로 교체:
1. 글은 6시간 후 자동으로 사라집니다
2. 좋아요 → 생명 연장, 싫어요 → 생명 단축
3. 매일 투표권 10개 무료 충전

**P0: CTA 문구 최적화 (A/B 테스트 대상)**

현재: "시작하기"
개선안:
- "지금 투표하기 — 투표권 10개 무료"
- "내 글 올리기 (6시간 생존 도전)"
- "생존 게임 참여하기"

**P1: 소셜 프루프 추가**
- 현재 소셜 프루프 없음
- 봇 활동 포함 누적 글 수 표시: "지금까지 [N]개의 글이 태어났습니다"
- 실시간 활성 글 수 배지: "현재 [N]개의 글이 살아있습니다"

**P1: FAQ 섹션 추가** (랜딩 하단 또는 별도 페이지)
- "글이 사라지면 어떻게 되나요?"
- "투표권이 떨어지면?"
- "유료 투표권은 왜 사야 하나요?"

#### A/B 테스트 로드맵

| 테스트 | 가설 | ICE |
|--------|------|-----|
| CTA 문구 변경 | "생존 게임 참여하기"가 클릭률 +20% | Impact:8 × Confidence:6 × Ease:9 = 432 |
| 3줄 룰 배너 | 컨셉 이해율 향상 → 회원가입률 +15% | Impact:8 × Confidence:7 × Ease:8 = 448 |
| 실시간 카운터 | FOMO 유발 → 참여율 +10% | Impact:6 × Confidence:6 × Ease:7 = 252 |

### 3-3. Product Hunt 런칭 전략

#### 런칭 전 준비 체크리스트

```
[ ] 트위터/X 계정 생성 + 팔로워 50명+ 확보 (2주 전)
[ ] Product Hunt 계정 생성 + Hunter 활동 (1주 전)
[ ] 런칭 갤러리 이미지 제작 5장 (1270×760px)
[ ] 태그라인 확정 (60자 이내): "The living anonymous bulletin board — posts survive or die by votes"
[ ] Maker 코멘트 초안 작성
[ ] 지지자 네트워크 사전 확보 (최소 30명)
[ ] 한국어 Product Hunt 유저 커뮤니티 탐색
```

#### 런칭 전략

**태그라인 후보**:
- "The bulletin board where posts live and die by votes"
- "Anonymous posts with a 6-hour lifespan — save them or let them die"
- "Posts have a pulse. Upvote to keep them alive, downvote to end them"

**Maker 코멘트 핵심 메시지**:
1. 왜 만들었나: "영구적으로 남는 인터넷 글에 피로감 → 사라지는 글 실험"
2. 어떻게 작동하나: 6시간 + 투표 +/-10분 메커니즘
3. 무엇을 발견했나: 소멸 임박 글에 몰리는 구조가 커뮤니티 역학 변화

**런칭 타이밍**: 화요일~목요일 (한국 시간 오후 1시 = 미국 PT 자정 기준)

**목표**: Top 5 of the Day

## 5. 성장 루프 설계

### PulseUp Core Growth Loop

```
신규 사용자 유입 (Acquisition)
  → 피드 탐색 → 소멸 임박 글 발견 → 투표 행동 (Activation Aha Moment)
  → 글 작성 → 생존 긴장감 경험 → 매일 재방문 확인 (Retention Hook)
  → "이 글 봐봐" 공유 → 지인 유입 (Referral Loop)
  → 무료 투표권 소진 → 유료 투표권 구매 (Revenue)
  → 게임성 강화 → 더 많은 활동 → 신규 사용자 추천 (Loop 반복)
```

### 바이럴 루프 트리거 아이디어

- "내 글 공유하기" 버튼 → 링크 클릭 시 잔여 생명력 표시
- "이 글이 X시간 후 사라집니다" OG 이미지 동적 생성
- "내 글 살려줘!" 공유 문구 자동 생성
